Statistical and ML-based fraud detection using Logistic Regression and Random Forest on 500 financial transactions
Author

Rohith Krishna Thazhathethil Chandraprakash

Published

March 1, 2025

Overview

This individual project examines a fraud detection dataset to identify patterns distinguishing fraudulent from genuine transactions. Using statistical methods and predictive modelling, it delivers a fraud mitigation strategy for upper management.

NoteDataset

500 transactions, 23 variables. Target: is_fraud. Nearly balanced — 246 fraud cases (49.2%) vs 254 legitimate (50.8%).

Analysis Pipeline

Fraud vs Non-Fraud Distribution

Code 
import plotly.express as px
import pandas as pd

df = pd.DataFrame({
    'Transaction Type': ['Non-Fraud (Class 0)', 'Fraud (Class 1)'],
    'Count': [254, 246]
})

fig = px.bar(df, x='Transaction Type', y='Count',
             title='Fraud vs Non-Fraud Distribution',
             color='Transaction Type',
             color_discrete_map={
                 'Non-Fraud (Class 0)': '#2980B9',
                 'Fraud (Class 1)': '#E74C3C'
             })
fig.update_layout(template='plotly_white', showlegend=False)
fig.show()

The dataset is balanced with 49.2% fraud and 50.8% legitimate transactions

Model Performance

Code 
import plotly.graph_objects as go

metrics = ['Accuracy', 'Fraud Recall', 'Fraud Precision', 'ROC-AUC']
logistic = [0.68, 0.74, 0.65, 0.675]
random_forest = [0.70, 0.75, 0.68, 0.692]

fig = go.Figure()
fig.add_trace(go.Bar(
    name='Logistic Regression',
    x=metrics, y=logistic,
    marker_color='#2980B9'
))
fig.add_trace(go.Bar(
    name='Random Forest',
    x=metrics, y=random_forest,
    marker_color='#8E44AD'
))

fig.update_layout(
    barmode='group',
    title='Logistic Regression vs Random Forest',
    yaxis=dict(range=[0.5, 0.85], title='Score'),
    template='plotly_white',
    legend_title='Model'
)
fig.show()

Random Forest outperforms Logistic Regression across all metrics

TipConclusion

Random Forest provides superior performance and is the preferred model for operational fraud detection. Fraud does not always appear in unusually large transactions — combining ML with statistical flagging (Z-score + IQR) gives the most robust detection approach.

Recommendations

  • Use Random Forest as the primary detection model
  • Implement real-time transaction risk scoring using model probability outputs
  • Prioritise awareness campaigns for older and lower-income customer segments
  • Combine Z-score and IQR anomaly checks with ML predictions for a layered defence
NoteStatistical Note

The IQR outlier capping method applied boundaries at \(Q1 - 1.5 \times IQR\) (lower) and \(Q3 + 1.5 \times IQR\) (upper) to Transaction Amount, Fraud Amount, Loan Amount, and Income variables.

## Analysis Pipeline

```{mermaid}
flowchart LR
  A[500 Transactions] --> B[Data Cleaning]
  B --> C[IQR Outlier Capping]
  C --> D[One-Hot Encoding]
  D --> E[75/25 Train-Test Split]
  E --> F[Z-Score & IQR Flagging]
  E --> G[Logistic Regression]
  E --> H[Random Forest 200 trees]
  F --> I[Insights]
  G --> I
  H --> I
```
